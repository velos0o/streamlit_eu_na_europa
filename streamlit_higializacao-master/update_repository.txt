# Instruções para Atualizar o Repositório

## Passo 1: Inicializar Git no Projeto Atual (se ainda não estiver inicializado)
```powershell
git init
```

## Passo 2: Adicionar Origem Remota
```powershell
git remote add origin https://github.com/velos0o/streamlit_higializacao.git
```

## Passo 3: Verificar o Status do Repositório
```powershell
git status
```

## Passo 4: Adicionar Arquivos ao Staging
```powershell
git add .
```

## Passo 5: Criar Commit com as Alterações
```powershell
git commit -m "Atualização completa do dashboard com proteção de credenciais"
```

## Passo 6: Forçar Atualização do Repositório Remoto
# ATENÇÃO: Isso irá sobrescrever completamente o repositório remoto
```powershell
git push -f origin master
# Ou se usar main como branch principal:
# git push -f origin main
```

## IMPORTANTE
1. O comando git push -f irá sobrescrever completamente o histórico do repositório remoto
2. Certifique-se de que o arquivo .env não está sendo enviado (verificado pelo .gitignore)
3. Configure as variáveis de ambiente no Streamlit Cloud após fazer o deploy

## Configurações do Streamlit Cloud
No painel de controle do Streamlit Cloud:
1. Acesse o app recém-implantado
2. Vá em Configurações > Secrets
3. Adicione as seguintes variáveis no formato TOML:
```toml
BITRIX_TOKEN = "seu_token_aqui"
BITRIX_URL = "sua_url_aqui"
BITRIX_REST_TOKEN = "seu_token_rest_aqui"
BITRIX_REST_URL = "sua_url_rest_aqui"
```
